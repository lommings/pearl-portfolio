# 疑難排解

常見問題與解決方案

---

## 安裝與環境問題

### Node.js 版本錯誤

**問題：**
```
Error: Astro requires Node.js v18.0.0 or higher
```

**解決：**
1. 下載最新 LTS 版本：https://nodejs.org
2. 驗證版本：`node -v`
3. 如果有多個版本，使用 nvm 切換：
   ```bash
   nvm install 18
   nvm use 18
   ```

---

### npm install 失敗

**問題：**
```
npm ERR! code EACCES
```

**解決（Windows）：**
1. 以管理員身分執行 PowerShell
2. 重新執行：`npm install`

**解決（macOS/Linux）：**
```bash
sudo chown -R $(whoami) ~/.npm
npm install
```

---

### Git 沒安裝

**問題：**
```
'git' is not recognized as an internal or external command
```

**解決：**
1. 下載安裝：https://git-scm.com
2. 重新啟動終端機
3. 驗證：`git --version`

## Astro 相關問題

### 本地開發伺服器無法啟動

**問題：**
```
Error: Cannot find module 'astro'
```

**解決：**
```bash
# 刪除 node_modules 重新安裝
rm -rf node_modules package-lock.json
npm install
```

---

### 圖片無法顯示

**問題：** 圖片路徑 404

**原因：** 靜態資源要放在 `public/` 資料夾

**正確做法：**
```
public/
└── images/
    └── my-image.jpg
```

在 Markdown 中：
```markdown
![描述](/images/my-image.jpg)
```

**使用 Astro Image 組件（推薦）：**
```
src/
└── assets/
    └── my-image.jpg
```

```astro
---
import { Image } from 'astro:assets';
import myImage from '../assets/my-image.jpg';
---

<Image src={myImage} alt="描述" width={800} />
```

---

### Markdown frontmatter 錯誤

**問題：**
```
Error: Invalid frontmatter
```

**常見錯誤：**
```markdown
---
title: 我的文章  ❌ (缺少引號)
date: 2026-02-24  ❌ (日期格式錯誤)
tags: 標籤1, 標籤2  ❌ (陣列格式錯誤)
---
```

**正確格式：**
```markdown
---
title: "我的文章"  ✅
pubDate: "Feb 24 2026"  ✅
tags: ["標籤1", "標籤2"]  ✅
---
```

---

### 熱重載（Hot Reload）不工作

**問題：** 修改檔案後網頁沒自動更新

**解決：**
1. 檢查檔案是否在 `src/` 資料夾內
2. 手動重新整理瀏覽器（Ctrl/Cmd + R）
3. 重啟開發伺服器：
   ```bash
   Ctrl + C
   npm run dev
   ```

---

### Build 失敗

**問題：**
```
npm run build
Error: The "path" argument must be of type string
```

**可能原因：**
- Markdown 檔案中有錯誤的圖片路徑
- frontmatter 資料格式錯誤
- 組件中有語法錯誤

**除錯步驟：**
1. 檢查終端機的錯誤訊息，找到問題檔案
2. 逐一檢查最近修改的檔案
3. 使用 `npm run dev` 測試（通常會顯示更詳細的錯誤）

## Git 與 GitHub 問題

### Push 被拒絕

**問題：**
```
error: failed to push some refs to 'https://github.com/...'
```

**解決：**
```bash
# 先拉取遠端變更
git pull origin main

# 解決衝突後再推送
git push origin main
```

---

### 帳號驗證問題

**問題：**
```
remote: Support for password authentication was removed
```

**解決（使用 Personal Access Token）：**
1. GitHub → Settings → Developer settings → Personal access tokens → Generate new token
2. 勾選 `repo` 權限
3. 複製 token
4. 推送時用 token 當密碼：
   ```bash
   Username: 你的帳號
   Password: ghp_xxxxxxxxxxxxx (token)
   ```

**或設定 SSH Key（推薦長期使用）：**
```bash
# 生成 SSH key
ssh-keygen -t ed25519 -C "your_email@example.com"

# 複製公鑰
cat ~/.ssh/id_ed25519.pub

# 貼到 GitHub → Settings → SSH and GPG keys → New SSH key
```

---

### .gitignore 不生效

**問題：** 已加入 `.gitignore` 的檔案還是被追蹤

**解決：**
```bash
# 清除快取
git rm -r --cached .
git add .
git commit -m "更新 .gitignore"
```

## Cloudflare Pages 部署問題

### 建置失敗

**問題：**
```
Build failed: Command failed with exit code 1
```

**常見原因與解決：**

1. **Node 版本不對**
   - Cloudflare Pages → Settings → Environment variables
   - 新增：`NODE_VERSION` = `18`

2. **建置指令錯誤**
   - 確認：`npm run build` 在本地可以成功
   - Cloudflare 設定應該是：`npm run build`

3. **輸出目錄錯誤**
   - Astro 預設輸出到 `dist/`
   - 確認 Cloudflare 設定：Build output directory = `dist`

4. **依賴項目問題**
   - 確保 `package.json` 的 `dependencies` 包含所有必要套件
   - 開發用的套件應該在 `devDependencies`

---

### 網站可以建置但無法訪問

**問題：** 部署成功但打開網址 404

**解決：**
1. 檢查 `astro.config.mjs` 的 `site` 設定
2. 確認 `package.json` 的 build script
3. 檢查 Cloudflare Pages 的建置日誌

---

### 自訂域名 SSL 錯誤

**問題：**
```
Your connection is not private
```

**解決：**
1. 等待 15-30 分鐘（SSL 憑證生效需要時間）
2. 確認 DNS 設定正確：
   - CNAME 記錄：`www` → `你的專案.pages.dev`
   - A 記錄（根域名）：指向 Cloudflare IP
3. Cloudflare → SSL/TLS → 設定為 "Full" 或 "Full (strict)"

## Waline 留言系統問題

### Vercel 部署失敗

**問題：** 一鍵部署卡住或失敗

**解決：**
1. 確認已登入 GitHub
2. 授權 Vercel 訪問 GitHub
3. 手動克隆 Waline repo：
   ```bash
   git clone https://github.com/walinejs/waline.git
   cd waline/example
   ```
4. 推到你的 GitHub，再從 Vercel import

---

### 留言顯示不出來

**問題：** 留言區空白或一直載入

**除錯步驟：**

1. **開啟瀏覽器 Console（F12）**
   - 檢查是否有錯誤訊息

2. **常見錯誤：**

   **CORS 錯誤：**
   ```
   Access to fetch at 'https://...' from origin '...' has been blocked by CORS
   ```
   **解決：** 在 Vercel 專案環境變數加入：
   ```
   SITE_URL=https://你的網站.com
   ```

   **Waline 伺服器無法連接：**
   ```
   Failed to fetch
   ```
   **解決：** 檢查 `serverURL` 設定是否正確

3. **手動測試 Waline API：**
   - 訪問：`https://你的waline.vercel.app/api/comment`
   - 應該返回 JSON 資料（不是錯誤頁面）

---

### 留言需要審核

**問題：** 留言送出後沒馬上顯示

**原因：** 預設需要管理員審核

**解決（關閉審核）：**
在 Vercel 環境變數加入：
```
AKISMET_KEY=
COMMENT_AUDIT=false
```

---

### 郵件通知不工作

**問題：** 有人留言但沒收到通知信

**檢查清單：**

1. **環境變數是否正確設定：**
   ```
   SMTP_SERVICE=Gmail
   SMTP_USER=你的email@gmail.com
   SMTP_PASS=應用程式密碼（16位，不是你的登入密碼）
   AUTHOR_EMAIL=你的email@gmail.com
   SITE_NAME=網站名稱
   SITE_URL=https://你的網站.com
   ```

2. **Gmail 應用程式密碼：**
   - Google 帳戶 → 安全性
   - 開啟「兩步驟驗證」
   - 「應用程式密碼」→ 生成新密碼
   - 複製 16 位密碼（去掉空格）

3. **檢查垃圾郵件資料夾**

4. **測試 SMTP：**
   - Vercel Function Logs 中檢查是否有錯誤

## 中國訪問問題

### 網站在中國載入很慢

**可能原因與解決：**

1. **使用了被牆的資源**
   - ❌ Google Fonts
   - ❌ Google Analytics
   - ❌ Disqus
   
   **替代方案：**
   - ✅ 本地字體或中國 CDN
   - ✅ Umami / Cloudflare Analytics
   - ✅ Waline 留言系統

2. **圖片太大**
   - 使用 Astro Image 自動優化
   - 或用線上工具壓縮（TinyPNG）

3. **沒用 CDN**
   - 確認使用 Cloudflare Pages（有中國節點）

### YouTube 影片不顯示

**問題：** YouTube 在中國被封鎖

**解決：** 使用 VideoPlayer 組件自動切換到 Bilibili

## 效能問題

### 網站載入慢

**診斷工具：**
- [PageSpeed Insights](https://pagespeed.web.dev)
- [WebPageTest](https://www.webpagetest.org)

**常見優化：**

1. **圖片優化**
   ```astro
   ---
   import { Image } from 'astro:assets';
   ---
   <Image src={...} width={800} format="webp" />
   ```

2. **懶載入**
   ```html
   <img src="..." loading="lazy" />
   <iframe src="..." loading="lazy"></iframe>
   ```

3. **移除未使用的 CSS/JS**
   - Astro 會自動做，但檢查是否引入太多外部庫

4. **啟用快取**
   - Cloudflare 自動處理，但可以在 Headers 設定：
   ```
   Cache-Control: public, max-age=31536000
   ```

---

### 建置時間太長

**問題：** `npm run build` 跑很久

**解決：**
1. 檢查是否有過多的圖片需要處理
2. 移除不必要的依賴：
   ```bash
   npm prune
   ```
3. 使用 `.astroignore` 排除不需要的檔案

## 留言系統進階問題

### 垃圾留言太多

**解決方案：**

1. **啟用 Akismet（垃圾留言過濾）：**
   - 註冊：https://akismet.com
   - 取得 API Key
   - Vercel 環境變數：
     ```
     AKISMET_KEY=你的key
     ```

2. **啟用驗證碼：**
   ```javascript
   Waline.init({
     // ...
     recaptchaV3Key: '你的reCAPTCHA key',
   });
   ```

3. **關鍵字過濾：**
   - 在 Waline 管理後台設定禁用詞

---

### 留言資料遺失

**預防措施：**

1. **定期備份**
   - Waline 管理後台 → 匯出留言（JSON）

2. **使用可靠的儲存**
   - LeanCloud（中國）
   - Supabase
   - MongoDB Atlas

3. **設定自動備份腳本**
   ```bash
   # cron job 每週備份
   0 0 * * 0 curl https://你的waline.vercel.app/api/export > backup.json
   ```

## 會員系統問題

### Supabase 連接失敗

**問題：**
```
Error: Invalid API key
```

**解決：**
1. 檢查 `.env` 檔案：
   ```
   PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
   PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

2. 確認 Astro 專案根目錄有 `.env`
3. 重新啟動開發伺服器

---

### 登入後馬上登出

**問題：** Session 無法保持

**解決：**
```typescript
// 確保使用正確的 storage
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  supabaseUrl, 
  supabaseKey,
  {
    auth: {
      persistSession: true,
      storageKey: 'my-app-auth',
    }
  }
);
```

## 其他常見問題

### 樣式沒有套用

**問題：** CSS 寫了但沒效果

**檢查：**
1. `<style>` 標籤是否在 Astro 組件內
2. 是否需要 `is:global`：
   ```astro
   <style is:global>
     /* 全局樣式 */
   </style>
   ```
3. CSS 選擇器是否正確

---

### 中文路徑問題

**問題：** 檔名或路徑包含中文導致錯誤

**建議：**
- ❌ `src/content/blog/我的文章.md`
- ✅ `src/content/blog/my-article.md`

在 frontmatter 用中文：
```markdown
---
title: "我的文章"  ✅
---
```

---

### TypeScript 錯誤

**問題：**
```
Type error: Cannot find module...
```

**解決（如果不用 TypeScript）：**
1. `tsconfig.json` → 設定 `"strict": false`
2. 或把 `.ts` 檔案改成 `.js`
