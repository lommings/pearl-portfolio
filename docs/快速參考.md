## 常用指令

### Astro 專案

```bash
# 建立新專案
npm create astro@latest my-site

# 安裝依賴
npm install

# 啟動開發伺服器（http://localhost:4321）
npm run dev

# 建置生產版本
npm run build

# 本地預覽建置結果
npm run preview

# 檢查專案問題
npm run astro check
```

---

### Git 操作

```bash
# 初始化 Git
git init

# 查看狀態
git status

# 加入所有變更
git add .

# 提交變更
git commit -m "說明訊息"

# 推送到 GitHub
git push origin main

# 拉取遠端變更
git pull origin main

# 查看提交歷史
git log --oneline

# 復原最後一次提交（保留檔案變更）
git reset --soft HEAD~1

# 放棄所有未提交的變更
git checkout .
```

---

### npm 套件管理

```bash
# 安裝套件
npm install 套件名稱

# 安裝開發用套件
npm install -D 套件名稱

# 移除套件
npm uninstall 套件名稱

# 更新所有套件
npm update

# 查看已安裝套件
npm list --depth=0

# 清除快取
npm cache clean --force

# 重新安裝所有依賴
rm -rf node_modules package-lock.json
npm install
```

## 環境變數範本

`.env` 檔案（記得加入 `.gitignore`）：

```env
# Supabase（如果使用會員系統）
PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Waline（如果自架）
WALINE_SERVER_URL=https://your-waline.vercel.app

# Google Analytics（可選）
PUBLIC_GA_ID=G-XXXXXXXXXX

# 其他 API Keys
PUBLIC_API_KEY=your_api_key_here
```

**安全注意：**
- `PUBLIC_` 開頭的變數會暴露在客戶端
- 敏感資訊（密碼、私鑰）不要用 `PUBLIC_` 開頭
- `.env` 檔案不要推送到 GitHub

## Astro.config.mjs 參考

```javascript
import { defineConfig } from 'astro/config';

export default defineConfig({
  // 網站 URL（部署後的網址）
  site: 'https://你的域名.com',
  
  // 輸出模式
  output: 'static', // 或 'server', 'hybrid'
  
  // 整合外掛
  integrations: [
    // 例如：sitemap(), mdx()
  ],
  
  // Markdown 設定
  markdown: {
    shikiConfig: {
      theme: 'github-dark', // 程式碼高亮主題
    },
  },
  
  // 圖片設定
  image: {
    domains: ['your-cdn.com'],
  },
  
  // Vite 設定（進階）
  vite: {
    build: {
      rollupOptions: {
        output: {
          // 自訂輸出
        },
      },
    },
  },
});
```

## 快速除錯檢查清單

遇到問題時依序檢查：

- [ ] 瀏覽器 Console 有沒有錯誤訊息（F12）
- [ ] 終端機有沒有錯誤訊息
- [ ] 檔案路徑是否正確
- [ ] 有沒有拼字錯誤
- [ ] 環境變數是否設定正確
- [ ] 重新啟動開發伺服器（Ctrl+C → npm run dev）
- [ ] 清除 node_modules 重裝（rm -rf node_modules && npm install）
- [ ] 檢查 Git 是否有未提交的變更
- [ ] 查看 Cloudflare/Vercel 的建置日誌
- [ ] Google 搜尋錯誤訊息

---

## 效能優化檢查清單

- [ ] 圖片使用 WebP 格式
- [ ] 圖片加上 `loading="lazy"`
- [ ] iframe 加上 `loading="lazy"`
- [ ] 移除未使用的 CSS/JS
- [ ] 使用 Astro Image 組件優化圖片
- [ ] 啟用 CDN（Cloudflare）
- [ ] 壓縮文字檔案（Astro 自動處理）
- [ ] 設定適當的快取標頭
- [ ] 使用系統字體或本地字體（避免 Google Fonts）

---

## 安全檢查清單

- [ ] `.env` 已加入 `.gitignore`
- [ ] 不要在前端暴露 API 密鑰
- [ ] 使用 `PUBLIC_` 前綴區分公開/私密變數
- [ ] CORS 設定正確
- [ ] 啟用 HTTPS（Cloudflare 自動）
- [ ] 定期更新依賴套件（`npm update`）
- [ ] 留言系統啟用垃圾過濾
- [ ] 會員密碼使用強加密（Supabase 自動處理）
